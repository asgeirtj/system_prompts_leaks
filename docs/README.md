# System Prompts Archive - Frontend

This `docs/` directory contains the completely static, vanilla HTML/JS frontend for the system prompts archive. 

It is designed to be hosted directly via GitHub Pages with **zero build steps** required.

## How it works

- The frontend dynamically fetches the archive data and files from GitHub.
- The index of available files is generated by `../scripts/build-site-index.mjs` and saved to `docs/data/index.json`.
- The UI uses vanilla JavaScript (ES modules) loaded from `assets/`.
- There is no Webpack, Vite, Node, or NPM required to run or edit this frontend.

## Regenerating the Index Locally

If you've added new `.md` files to the repository and want them to show up in the frontend during local testing:

```bash
node scripts/build-site-index.mjs
```

## Running Locally

Because the frontend uses ES modules, you must serve it over HTTP (opening `index.html` directly in a browser via `file://` will not work).

From the root of the repository, you can use any static server:

```bash
python -m http.server --directory docs
```

Then open `http://localhost:8000/`.

## Modifying the UI

You can edit the files in `docs/assets/` directly. No compilation step is required. Just refresh your browser.
